# Редактор базы знаний (`knowledge_base.json`)

React-приложение (Vite + TypeScript) для правки структуры базы знаний аватара: псевдонимы продуктов, пары Q&A, шаблоны `product_template`. Блок `recommendation_rules` остаётся в JSON и правится только в режиме «Сырой JSON». Изменения вносятся **только в этом каталоге**; корень монорепозитория не затрагивается.

## Локально

```bash
npm install
npm run sync-kb   # скопировать ../knowledge_base.json → public/ (если файл есть)
npm run dev
```

Если `public/knowledge_base.json` отсутствует или устарел, выполните `sync-kb` или импортируйте JSON через кнопку «Импорт JSON».

## Сборка

```bash
npm run build
npm run preview
```

## Деплой на Vercel

1. Подключите репозиторий к [Vercel](https://vercel.com/).
2. В настройках проекта задайте **Root Directory**: `data_collection`.
3. Подключите **Vercel Blob**: в проекте Vercel → **Storage** → **Create Database** → **Blob** (или через вкладку Blob), привяжите store к этому проекту. В переменные окружения добавится **`BLOB_READ_WRITE_TOKEN`** (можно сгенерировать/скопировать в настройках store).

4. Сохраните остальные переменные (Settings → Environment Variables):

| Переменная | Назначение |
|------------|------------|
| `BLOB_READ_WRITE_TOKEN` | Обычно подставляется автоматически при подключении Blob к проекту; для `vercel dev` выполните `vercel env pull`. |
| `KB_UPLOAD_TOKEN` | Ваш секрет для вызова `POST /api/upload-kb` (тот же токен вводится в UI редактора). |
| `BLOB_OBJECT_KEY` | Опционально: путь объекта в Blob (по умолчанию `articles/knowledge_base.json`, как в примере `put('articles/...')`). |
| `VITE_PUBLIC_KB_URL` | Опционально: **публичный** URL загруженного JSON (`access: 'public'`). После сборки приложение сначала пытается загрузить базу отсюда. См. `.env.example`. |

### Публичная «база» на Blob

Объект кладётся через `put(..., { access: 'public' })` — по ссылке JSON можно читать из браузера (в том числе этим редактором). Порядок загрузки при открытии сайта:

1. URL из поля «Публичный URL» в UI (сохраняется в `localStorage`, ключ `kb_public_url`);
2. иначе `VITE_PUBLIC_KB_URL` из переменных окружения при **сборке**;
3. иначе файл `public/knowledge_base.json`.

После успешной отправки в Blob редактор сам подставляет возвращённый `url` и обновляет данные.

Загрузка с локального `npm run dev` **не** попадёт в `/api/*` (Vite). Для проверки API используйте деплой на Vercel или `vercel dev` из каталога `data_collection` (с подтянутым `BLOB_READ_WRITE_TOKEN`).

## Выгрузка и возврат в основной проект

1. **Скачать JSON** — сохраняет текущее состояние в `knowledge_base.json`.
2. Вручную замените файл `knowledge_base.json` в **корне** репозитория аватара (вне `data_collection`) и выполните переиндексацию Chroma по инструкции в `doc/knowledge_base_guide.md` в основном проекте.

Автоматические скрипты в корне репозитория для этого не добавляются.

## Безопасность

- Не коммитьте `KB_UPLOAD_TOKEN` и `BLOB_READ_WRITE_TOKEN` в репозиторий.
- Токен в браузере хранится в `localStorage` (ключ `kb_upload_token`) для удобства; на общих машинах вводите его заново и не сохраняйте.
- Публичный URL Blob — тоже в `localStorage` (`kb_public_url`); по этой ссылке **любой** может скачать JSON. Не публикуйте её там, где нельзя светить базу знаний.

## Ограничения

- Размер тела запроса на Vercel ограничен (типично ~4.5 MB на Hobby); очень большой JSON может потребовать другой способ загрузки.
- Схема валидации (Zod) описана в `src/kbSchema.ts`; при расширении формата основного `knowledge_base.json` при необходимости обновите схему здесь.
